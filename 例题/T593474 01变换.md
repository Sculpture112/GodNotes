# []结尾有对于前缀异或和的解答

> **原题链接:** (https://www.luogu.com.cn/problem/T593474)

**涉及知识点:** [[异或]], [[前缀异或和]]，[[待补题]],[[]],[[]]

**核心套路:** 


```cpp
#include <iostream>
#include <string>
#include <vector>

using namespace std;

void solve() {
    int n, m;
    cin >> n >> m;
    string s, t;
    cin >> s >> t;

    if (n < m) {
        cout << "NO\n";
        return;
    }

    // 计算前缀异或和
    vector<int> ps(n + 1, 0), pt(m + 1, 0);
    for (int i = 0; i < n; ++i) ps[i + 1] = ps[i] ^ (s[i] - '0');
    for (int i = 0; i < m; ++i) pt[i + 1] = pt[i] ^ (t[i] - '0');

    // 整个字符串的总异或和必须相等
    if (ps[n] != pt[m]) {
        cout << "NO\n";
        return;
    }

    // 贪心匹配前缀异或和子序列
    int ptr = 1;
    for (int i = 1; i < n; ++i) {
        if (ptr < m && ps[i] == pt[ptr]) {
            ptr++;
        }
    }

    // 如果成功匹配了 m-1 个分割点
    if (ptr == m) {
        cout << "YES\n";
    } else {
        cout << "NO\n";
    }
}

int main() {
    // 优化输入输出速度
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    solve();
    return 0;
}
```

---

这其实是利用了异或运算（XOR，符号为 $\oplus$）的一个非常神奇的性质：**“自我抵消”**。

简单来说，这个公式是**前缀和**思想在异或运算上的应用。

### 1. 异或运算的关键性质

在理解公式之前，要记住异或的两个“超能力”：

1. **$x \oplus x = 0$**（任何数异或自己都等于 0）
    
2. **$x \oplus 0 = x$**（任何数异或 0 保持不变）
    

### 2. 拆解前缀异或和 $ps$

我们定义 $ps[i]$ 为原序列前 $i$ 个元素的异或总和：

- $ps[0] = 0$
    
- $ps[1] = s_0$
    
- $ps[2] = s_0 \oplus s_1$
    
- $ps[3] = s_0 \oplus s_1 \oplus s_2$
    
- 以此类推...
    

### 3. 公式推导：为什么要“减去”前面的部分？

假设我们想要得到区间 $[l, r]$（下标从 $0$ 开始）的异或和，即：

$$\text{目标} = s_l \oplus s_{l+1} \oplus \dots \oplus s_r$$

我们可以观察 $ps[r+1]$ 的组成：

$$ps[r+1] = \underbrace{(s_0 \oplus s_1 \oplus \dots \oplus s_{l-1})}_{\text{前 } l \text{ 个元素}} \oplus \underbrace{(s_l \oplus \dots \oplus s_r)}_{\text{我们要的区间}}$$

你会发现，$ps[r+1]$ 其实就是 **“前 $l$ 个元素的异或和”** 加上 **“我们要的区间异或和”**。

即：$ps[r+1] = ps[l] \oplus \text{区间异或和}$

现在，我们在等式两边同时**异或上 $ps[l]$**：

$$ps[r+1] \oplus ps[l] = (ps[l] \oplus \text{区间异或和}) \oplus ps[l]$$

根据交换律和“自我抵消”性质：

$$ps[r+1] \oplus ps[l] = (ps[l] \oplus ps[l]) \oplus \text{区间异或和}$$

$$ps[r+1] \oplus ps[l] = 0 \oplus \text{区间异或和}$$

$$ps[r+1] \oplus ps[l] = \text{区间异或和}$$

### 4. 直观理解

这就像是在做**减法**。

- 在普通加法前缀和中：区间和 = $sum[r+1] - sum[l]$。
    
- 在异或运算中：**“异或”操作本身就是它自己的逆运算**（相当于加法里的减法）。
    

所以，当我们用“前 $r+1$ 个人的总和”去“异或”掉“前 $l$ 个人的总和”时，前 $l$ 个人因为被异或了两次，自动消失（变成 $0$）了，剩下的就是中间那段 $[l, r]$ 的结果。

这就是为什么在算法题中，前缀异或和是处理区间查询的神器，因为它能把 $O(n)$ 的查询优化到 $O(1)$。
