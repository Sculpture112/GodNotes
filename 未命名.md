- **第 056-057 节：并查集**
    
    - **为什么重要：** 省赛极其喜欢考“图的连通性”、“朋友圈”、“分阵营”的问题，全靠并查集。核心代码就三个函数（初始化、查找祖先、合并），五分钟就能背下来。
        
- **第 061 节：最小生成树（重点练 Kruskal 算法）**
    
    - **为什么重要：** 只要题目让你“修路连接所有城市，求最少花费”，就是最小生成树。Kruskal 算法的本质就是“把边排序 + 并查集”。你掌握了排序和并查集，这个算法白送。
        
- **第 058 节：洪水填充（Flood Fill）**
    
    - **为什么重要：** 走迷宫、扫雷、计算岛屿数量等网格类题目必考。本质就是最简单的图遍历（DFS 或 BFS），是省赛签到题的常客。
        
- **第 065 节中的 Floyd 算法**
    
    - **为什么重要：** 求多源最短路。代码只有极其简单的三层 `for` 循环，遇到 $N \le 300$ 的最短路问题，闭着眼睛直接敲 Floyd，稳拿分。
        
- **第 070-071 节：子数组最大累加和**
    
    - **为什么重要：** 最基础的线性 DP / 贪心思想，代码只需要一个循环，极其经典，常作为复杂题目的一个内部步骤。
        

### ⭐ T1 级别：中档题主力（需要花时间刷题巩固）

这部分是你冲击省赛铜牌甚至银牌的底气。理解起来有些难度，需要通过刷题来适应各种变形。

- **第 064 节：Dijkstra 算法（单源最短路）**
    
    - **注意点：** 重点掌握配合优先队列（`priority_queue`）的优化版本。不要去纠结“分层图最短路”（那是 T2 级别），先把最基础的“从起点到终点的最短距离”写熟练。
        
- **第 062 节：BFS 及其扩展**
    
    - **注意点：** BFS（广度优先搜索）求无权图的最短步数是绝对的重点。必须熟练使用 `std::queue` 来写。
        
- **第 066-068 节：一维、二维动态规划（DP）**
    
    - **注意点：** DP 是算法竞赛的重头戏。这两节是带你入门的，重点去体会“状态定义”和“状态转移方程”是怎么推导的。去洛谷刷几道最简单的递推和线性 DP 题。
        
- **第 072 节：最长递增子序列（LIS）**
    
    - **注意点：** 标准 DP 模型。知道 $O(N^2)$ 的 DP 怎么写，尽量掌握 $O(N \log N)$ 的二分优化版本。
        
- **第 059 节：建图、拓扑排序**
    
    - **注意点：** 遇到“选课顺序”、“任务依赖”类的题目必用。关于建图，如果觉得“链式前向星”太难写、老写错，**强烈建议直接用 C++ 的 `vector<vector<int>>`（邻接表）建图**，对于省赛绝大多数题目完全够用，且不容易出错！
        

### 🛑 T2 级别：战略性放弃（目前不要看，考场上遇到直接跳）

在距离省赛只剩不到四周的情况下，看这些属于“投入极高、产出极低”，因为你即使看懂了，在赛场上也极容易写挂。

- **第 063 节：双向广搜**（普通的 BFS 够用了，双向广搜写错的概率太高）。
    
- **第 064 节后半段：分层图最短路**（难度超标，遇到了直接放弃）。
    
- **第 065 节：A星算法、SPFA、Bellman-Ford**（A星几乎用不到，最短路用 Dijkstra 和 Floyd 就能解决 95% 的问题。SPFA 容易被出题人卡时间（常说的“关于 SPFA，它死了”），初学者先保 Dijkstra）。
    
- **第 069 节：三维动态规划**（太复杂，把一维和二维弄明白已经很棒了）。
