## 💡 破题切入点 (思维闪念)

> [!tip] 核心直觉
> 1. 看到“子数组和”立马想到 **【前缀和】**。
> 2. 发现数组包含负数，说明前缀和数组不再单调递增。这意味着 **不能用普通的双指针滑动窗口**（因为右移左指针不一定会让区间和变小，窗口的扩大/缩小失去了单调性）。
> 3. 问题转化为：对于当前的右端点 `j`，如何在 `j` 之前找到最大的 `i`，使得 `sum[j] - sum[i] >= k`？

> [!info] 状态淘汰法则（为什么用单调队列？）
> 寻找过程中发现无效状态可以舍弃，完美契合**单调队列**：
> - **没用的高个子（维持单调性）**：如果前面的前缀和比当前的前缀和还大，它作为减数更吃亏，且距离还更远，直接从队尾淘汰（维持队列单调递增）。
> - **用完即废的队头（求最短距离）**：一旦队头跟当前的 `j` 匹配成功，算完长度就可以扔了。因为它跟以后的 `j` 匹配只会产生更长的子数组，而我们要求的是最短。

---

## ⚙️ 核心套路与瓶颈

> [!abstract] 核心套路
> 1. **前缀和转换**：将子数组和 `>= K` 转换为寻找 `sum[j] - sum[i] >= K`。
> 2. **队头结算**：若当前前缀和与队头前缀和之差 `>= K`，更新答案并弹出队头（因后续更靠右的 `j` 即使达标，距离 `i` 的长度也会更长，`i` 已失去利用价值）。
> 3. **队尾淘汰**：若当前前缀和 `<=` 队尾前缀和，弹出队尾（因当前元素前缀和更小且位置更靠右，作为“被减数”在未来更容易满足 `>= K` 的条件且构成的子数组更短，具有对队尾的“绝对优势”）。

> [!warning] 时空限制瓶颈
> - 时间复杂度必须优化至 `O(N)` 或 `O(NlogN)`，暴力 `O(N^2)` 会导致 TLE。
> - 数组元素可能很大或有负数，累加过程会超过 32 位有符号整数上限，**前缀和数组必须使用 `long long` 防止溢出**。

## 💻 核心代码 (C++)

```cpp
#include <vector>
#include <algorithm>
#include <climits>

using namespace std;

class Solution {
public:
    int shortestSubarray(vector<int>& arr, int k) {
        int n = arr.size();
        
        // sum[i] : 前 i 个数的前缀和
        // 使用 long long 防止累加过程中溢出
        vector<long long> sum(n + 1, 0);
        for (int i = 0; i < n; i++) {
            sum[i + 1] = sum[i] + arr[i];
        }
        
        // deque: 模拟双端队列，存储 sum 数组的索引
        // 局部开辟空间，随生命周期自动销毁
        vector<int> deque(n + 1);
        int h = 0, t = 0; // 头尾指针
        int ans = INT_MAX;
        
        // 遍历所有前缀和（包括前 0 个数的前缀和）
        for (int i = 0; i <= n; i++) {
            // 1. 检查队头，如果当前前缀和 - 队头前缀和 >= k，达标！
            // 结算并弹出队头，因为更长距离的没必要再看了
            while (h != t && sum[i] - sum[deque[h]] >= k) {
                ans = min(ans, i - deque[h++]);
            }
            
            // 2. 维护单调递增队列（小 -> 大）
            // 如果队尾的前缀和 >= 当前前缀和，队尾淘汰
            while (h != t && sum[deque[t - 1]] >= sum[i]) {
                t--;
            }
            
            // 3. 当前前缀和索引从尾部加入
            deque[t++] = i;
        }
        
        return ans != INT_MAX ? ans : -1;
    }
};
